# FreeCAD-generate-build

目标在于不依赖FreeCAD官方LibPack的情况下，产生visual studio的工程项目。

## 预备环境
1. 根目录之上的，同级目录存在 FreeCAD 源码，并被命名为 FreeCAD-git-fork
2. 同级目录存在 FreeCAD-LibPack，按 boost, python, eigen3 分子文件夹的模式组织依赖库
3. 执行 buildscript_vs2017.bat 启用cmake生成工程，此工程存放在本目录之下的 Debug
4. 调用 vs2017 打开工程，需要通过 msvc_vs2017.bat 来执行
